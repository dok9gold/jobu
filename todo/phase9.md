# 깃 정리 및 코어 분리, 템플릿(샘플, 뷰디스패치) 프렌치 생성, 구현
1. 현재 개발소스 별도 브렌치에 보관 (개발 테스트가 완료 된 버전)
2. 코어만 분리하여 메인브렌치에 저장, 히스토리 삭제
3. pip 등록 가능 한 수준으로 변경
4. 코어 분리 후 완성되노 메인브렌치에 (1)에서 저장한 브렌치를 머지해서 sample 브렌치 생성
    - sample 브렌치는 기본 워커 작성법, 다중디비 활용법, 비동기 디비 활용법 등의 워커 코딩가이드 제공
    - 기존에 만들어져 있는 9개 샘플은 정리 후 재 작성 예정
5. 메인 브렌치에서 큐 디스페치 브렌치 생성
    - 큐 디스패치 모듈 추가
        - 도커 컴포즈에 카프카 추가
        - admin 에 큐 발행 화면 및 api 추가
